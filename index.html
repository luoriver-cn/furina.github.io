Hexo文件目录结构

发表于 2025-03-15
本文字数： 1.4k 阅读时长 ≈ 5 分钟

根目录

  • .github：通常用于存放与 GitHub 相关的配置文件，比如 GitHub Actions 的工作流文件。

    • dependabot.yml： Dependabot 配置文件，用于自动更新项目依赖。

  • node_modules：存放项目所依赖的 Node.js 模块，这些模块是 Hexo 运行和构建项目所必需的。

  • public：这是 Hexo 生成的静态文件输出目录，默认情况下，当您运行 hexo generate 命令时，Hexo 会将生成的静态文件（如 HTML、CSS、JavaScript 等）放置在这里。这些文件可以直接部署到服务器上。

    • 2025：按年份组织的文件夹，用于存放特定年份的相关文件。
    • archives：存档页面相关文件。
    • css：样式表文件。
    • fancybox：一个用于实现图片灯箱效果的插件相关文件。
    • js：JavaScript 文件。
    • tags：标签页面相关文件。
    • atom.xml：网站的 Atom 格式订阅源文件。
    • index.html：网站的首页文件。

  • scaffolds：存放模板文件，当使用 hexo new 命令创建新文章时，Hexo 会根据该目录下的模板文件来生成新的 Markdown 文件。默认包含 post.md、page.md、draft.md 等模板。

    • draft.md：草稿模板，用于生成草稿文章。
    • page.md：页面模板，用于生成非文章页面如关于页面。
    • post.md：文章模板，用于生成普通博客文章。

  • source：这是存放用户原始内容的目录，所有的文章、页面等 Markdown 文件都应该放置在这里。Hexo 会根据这些文件生成静态页面。

    • _posts：文章目录，存放所有的博客文章 Markdown 文件。

  • themes：存放主题相关的文件，Hexo 的主题决定了博客的外观和样式。每个主题通常都有自己的目录，里面包含了布局文件、样式表、脚本等。

    • .gitkeep：一个空文件，用于确保该目录在版本控制系统中被保留。

  • _config.landscape.yml：这是 Hexo 的一个配置文件，通常用于一些特定的配置，比如 Landscape 主题的配置。不过在这个例子中，文件大小为 0 KB，可能是一个空文件或者未被使用。

  • _config.yml：这是 Hexo 的主配置文件，包含了博客的基本设置，如站点标题、描述、URL、文章排序方式、主题配置等。用户可以通过修改这个文件来定制博客的行为和外观。

  • db.json：这是 Hexo 的数据库文件，用于存储文章的元数据和其他相关信息。它有助于 Hexo 快速地生成和更新静态文件。

  • package.json：这是一个标准的 Node.js 项目配置文件，定义了项目的元数据、依赖项、脚本等。通过运行 npm install 命令，可以安装项目所需的依赖项。

  • package-lock.json：这是 npm 自动生成的文件，用于锁定项目依赖的具体版本，确保在不同环境下安装的依赖版本一致，避免因版本差异导致的问题。

自定义 scaffolds

  1. 修改默认模板：
    • 找到你的 Hexo 项目目录下的 scaffolds 文件夹。
    • 在这个文件夹中，你可以看到几个默认的模板文件，如 post.md、page.md、draft.md。
    • 直接编辑这些文件，修改它们的内容来改变新创建文章的默认格式。例如，你可以在 post.md 中添加一些默认的元数据、开头内容或者特定的格式设置。
  2. 创建新的模板类型：
    • 如果你需要一种特定类型的文章模板（比如技术文章、读书笔记等），可以在 scaffolds 文件夹中创建一个新的 Markdown 文件，例如 tech_post.md。
    • 在这个新文件中，定义你想要的模板内容，包括标题、日期、标签等元数据，以及任何你希望在新文章中默认出现的内容。
    • 当你需要创建这种类型的文章时，使用命令 hexo new tech_post "文章标题"，Hexo 就会根据这个模板生成新的文章文件。

自定义 themes

  1. 安装和切换主题：
    • 首先，你需要获取一个 Hexo 主题。可以通过 npm 安装，例如 npm install hexo-theme-ayer --save，或者使用 Git 克隆主题仓库到 themes 文件夹中，如 git clone https://github.com/Pcrab/hexo-theme-quark.git themes/quark。
    • 安装完成后，在 Hexo 项目的根目录下找到 _config.yml 文件，修改其中的 theme 参数，将其值设置为你安装的主题名称，例如 theme: ayer 或 theme: quark。
  2. 配置主题参数：
    • 每个主题通常都有自己的配置文件，位于 themes/[主题名称]/_config.yml。打开这个文件，根据主题的文档说明修改其中的参数，以实现个性化的外观和功能设置。例如，你可以修改主题的颜色方案、字体、布局等。
    • 有些主题可能还提供额外的配置选项，比如在主题的 _config.yml 中设置导航栏菜单项、侧边栏显示内容等。
  3. 自定义主题文件：
    • 如果需要更深入的自定义，比如修改主题的布局、样式或脚本，你可以直接编辑主题文件夹中的相关文件。例如，在 layout 文件夹中修改模板文件，在 source 文件夹中添加或修改资源文件（如 CSS、JavaScript）。
    • 不过，在直接修改主题文件之前，建议先备份原始文件，或者使用主题提供的扩展机制（如果有的话），以免在主题更新时丢失自定义的更改。

文件操作

  • 新建文件：Ctrl+Alt+N（Windows）/ Cmd+N（macOS）
  • 打开文件：Ctrl+Shift+N（Windows）/ Cmd+Shift+O（macOS）
  • 保存文件：Ctrl+S（Windows）/ Cmd+S（macOS）
  • 保存所有文件：Ctrl+Alt+S（Windows）/ Cmd+Option+S（macOS）
  • 关闭文件：Ctrl+F4（Windows）/ Cmd+W（macOS）

编辑操作

  • 撤销：Ctrl+Z（Windows）/ Cmd+Z（macOS）
  • 重做：Ctrl+Y（Windows）/ Cmd+Y（macOS）
  • 剪切：Ctrl+X（Windows）/ Cmd+X（macOS）
  • 复制：Ctrl+C（Windows）/ Cmd+C（macOS）
  • 粘贴：Ctrl+V（Windows）/ Cmd+V（macOS）
  • 删除：Delete（Windows）/ Delete（macOS）

代码格式化

  • 格式化代码：Ctrl+Alt+L（Windows）/ Cmd+Option+L（macOS）
  • 自动完成代码：Ctrl+Space（Windows）/ Cmd+Space（macOS）
  • 显示代码完成：Ctrl+Shift+Space（Windows）/ Cmd+Shift+Space（macOS）

导航

  • 跳转到文件：Ctrl+Shift+N（Windows）/ Cmd+Shift+O（macOS）
  • 跳转到类：Ctrl+N（Windows）/ Cmd+O（macOS）
  • 跳转到符号：Ctrl+Alt+Shift+N（Windows）/ Cmd+Option+Shift+O（macOS）
  • 跳转到行：Ctrl+G（Windows）/ Cmd+L（macOS）
  • 返回上一个位置：Ctrl+Alt+Left（Windows）/ Cmd+Option+Left（macOS）
  • 前进到下一个位置：Ctrl+Alt+Right（Windows）/ Cmd+Option+Right（macOS）

调试

  • 启动调试：Shift+F9（Windows）/ Cmd+F9（macOS）
  • 调试选中代码：Ctrl+Shift+F9（Windows）/ Cmd+Shift+F9（macOS）
  • 暂停调试：F8（Windows）/ F8（macOS）
  • 步过：F8（Windows）/ F8（macOS）
  • 步入：F7（Windows）/ F7（macOS）
  • 步出：Shift+F7（Windows）/ Shift+F7（macOS）

运行

  • 运行应用程序：Shift+F10（Windows）/ Cmd+R（macOS）
  • 运行选中代码：Ctrl+Shift+F10（Windows）/ Cmd+Shift+R（macOS）
  • 查看运行配置：Ctrl+Shift+F10（Windows）/ Cmd+Shift+R（macOS）

项目操作

  • 打开项目设置：Ctrl+Alt+S（Windows）/ Cmd+Option+S（macOS）
  • 打开项目结构：Ctrl+Shift+Alt+S（Windows）/ Cmd+Shift+Option+S（macOS）
  • 打开模块设置：Ctrl+Alt+Shift+S（Windows）/ Cmd+Option+Shift+S（macOS）

Typora

发表于 2025-03-15
本文字数： 1.1k 阅读时长 ≈ 4 分钟

Typora 是一款流行的 Markdown 编辑器，以其简洁的界面和高效的写作体验而受到许多用户的喜爱。以下是一些常见的 Typora 使用方法和技巧：

1. 安装与启动

  • 下载与安装：访问 Typora 的官方网站 https://www.typora.io/，根据你的操作系统（Windows、macOS 或 Linux）下载相应的安装包并安装。
  • 启动：安装完成后，双击 Typora 图标启动编辑器。

2. 界面介绍

  • 编辑区：主界面中央的空白区域，用于输入和编辑文本。
  • 菜单栏：顶部的菜单栏提供了各种操作选项，如文件操作、编辑、视图、插入等。
  • 格式栏：在某些视图模式下，顶部或侧面会有格式工具栏，提供快速格式化按钮。

3. 基本操作

  • 新建文件：点击菜单栏的 文件 > 新建 或使用快捷键 Ctrl+N（Windows）/ Cmd+N（macOS）。
  • 打开文件：点击 文件 > 打开 或使用快捷键 Ctrl+O（Windows）/ Cmd+O（macOS）。
  • 保存文件：点击 文件 > 保存 或使用快捷键 Ctrl+S（Windows）/ Cmd+S（macOS）。Typora 支持多种文件格式，如 .md、.txt、.html 等。
  • 导出文件：点击 文件 > 导出，可以选择导出为 PDF、HTML、DOCX 等格式。

4. 文档编辑

  • 输入文本：直接在编辑区输入文本内容。
  • 标题：使用 # 符号创建标题，# 的数量表示标题级别（1 到 6 级）。
  • 段落：直接输入文本即可创建段落，段落之间需要有一个空行。
  • 强调：使用 * 或 _ 表示斜体，** 表示粗体，*** 表示粗体斜体。
  • 列表：
    • 无序列表：使用 *、+ 或 -。
    • 有序列表：使用数字后跟点 .。
  • 链接：使用方括号 [] 包裹链接文本，然后使用圆括号 () 包裹链接地址。
  • 图像：与链接类似，但在前面添加一个感叹号 !。
  • 引用：使用 > 符号创建块引用。
  • 代码块：使用反引号 ` 包裹代码，或者使用缩进（4 个空格或一个制表符）。

5. 视图模式

  • 写作模式：点击菜单栏的 视图 > 写作模式，隐藏所有工具栏和侧边栏，专注于文本编辑。
  • 阅读模式：点击 视图 > 阅读模式，以只读模式查看文档。
  • 源代码模式：点击 视图 > 源代码模式，显示原始 Markdown 代码。

6. 插入内容

  • 插入链接或图像：点击菜单栏的 插入 > 链接 或 插入 > 图像，或者使用快捷键 Ctrl+L（Windows）/ Cmd+L（macOS）和 Ctrl+Alt+I（Windows）/ Cmd+Option+I（macOS）。
  • 插入表格：点击 插入 > 表格，选择表格的行列数。
  • 插入代码块：点击 插入 > 代码块，选择编程语言。
  • 插入数学公式：点击 插入 > 数学块 或 插入 > 行内数学，输入 LaTeX 格式的数学公式。

7. 自定义设置

  • 主题设置：点击菜单栏的 Typora > 设置（macOS）或 工具 > 选项（Windows/Linux），在 外观 选项卡中选择或自定义主题。
  • 快捷键设置：在设置中可以自定义快捷键，提高编辑效率。
  • 语法高亮：在设置中启用语法高亮，使代码块和数学公式等更易于阅读。

8. 同步与备份

  • iCloud 同步（macOS）：在设置中启用 iCloud 同步，跨设备同步文档。
  • 备份文档：定期使用 文件 > 导出 功能备份重要文档。

9. 实用技巧

  • 快速格式化：使用格式栏按钮或快捷键快速应用标题、列表、引用等格式。
  • 实时预览：Typora 的一大特点是实时渲染，输入时即可看到格式化效果，无需切换预览模式。
  • 多窗口编辑：可以同时打开多个文档窗口，方便对比和编辑。
  • 搜索与替换：使用快捷键 Ctrl+F（Windows）/ Cmd+F（macOS）打开搜索框，进行文本搜索与替换。

Typora 的简洁界面和丰富功能使其成为 Markdown 编辑的首选工具之一。通过熟悉上述使用方法，可以更高效地进行文档编写和管理。

Hexo GUI 介绍

概述

Hexo.app 是一个基于 PyQt5 的图形化用户界面（GUI）工具，专为简化 Hexo 博客管理系统以及网络端口转发等操作而设计。它提供了直观的多标签界面，让用户能够方便地管理博客、执行命令、监控进程以及处理网络配置等任务。

核心功能

1. Hexo 管理模块

2. NAT 控制模块

3. 多进程监控模块
